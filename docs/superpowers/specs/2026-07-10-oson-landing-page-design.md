# 오손(OSON) 홍보 랜딩페이지 설계

날짜: 2026-07-10

## 목적
한강·수상레저 사업장 사장님들에게 SaaS(브랜드명: 오손 OSON)를 소개하고
도입 문의 리드를 확보하는 B2B 랜딩페이지. 모바일 호환 필수.

## 배경 결정 사항
- 타겟: 사업장 사장님 (B2B). 손님 대상 매장 홍보 아님.
- CTA: 도입 문의 폼 (이름·연락처·사업장명·문의내용). 온보딩이 수동이라 상담 필수 — 리드 확보가 핵심.
- 가격: 비공개 ("문의 시 안내"). 유료화 전이라 유연성 유지.
- 브랜드: 오손(OSON) — "오! 손님" + "오손도손" + ocean 발음 유사. 업종 확장 가능.
- 제작 방식: 하이브리드 — 힉스필드(Higgsfield) create_website로 디자인 초안 생성 후
  코드를 기존 서버로 이식. 호스팅·데이터는 기존 인프라 사용.

## 아키텍처
- 기존 Railway FastAPI 앱이 서빙. `GET /`의 `/admin/` 리다이렉트를 랜딩페이지 응답으로 교체.
  `/admin/` 대시보드·기존 라우트는 변경 없음.
- 새 라우터 `app/routers/landing.py`:
  - `GET /` — 랜딩 HTML 서빙 (정적 파일, 템플릿 엔진 불필요)
  - `POST /api/leads` — 문의 폼 접수
- 랜딩 자산: `static/landing/` (HTML은 파일로 읽어 서빙하거나 FileResponse)
- Supabase `leads` 테이블 신설:
  `id, name, phone, business_name, message, created_at` (+ RLS 활성화, service_role 접근)
- 접수 성공 시 기존 슬랙 웹훅으로 알림 → 홍보전략가 루틴이 주간 리드 수 추적 가능

## 페이지 구성 (모바일 퍼스트 반응형, 단일 페이지)
1. 히어로: "오! 손님 — 사장님 대신 손님을 받는 AI 운영 시스템" + [도입 문의] 버튼(폼으로 스크롤)
2. 문제 공감: 성수기 카톡 폭주 / 노쇼 / 강습 중 응대 불가
3. 핵심 기능 5종: 카톡 AI 즉답(5초) · 예약/잔여석 · 노쇼 방지 · 날씨 안내 · 사진앨범
4. 도입 사례: "광나루 서울윈드서핑장 서퍼스트 운영 중" (성과 숫자는 확보되는 대로 추가)
5. 문의 폼: 이름 / 연락처 / 사업장명 / 문의내용 — 가격은 "문의 시 안내" 표기

## 스팸·안정성
- honeypot 숨김 필드 (봇이 채우면 조용히 무시)
- IP당 시간당 제출 제한 (인메모리 카운터면 충분)
- 랜딩 라우터는 카톡 웹훅과 완전 분리 — 랜딩 오류가 응대에 전파되지 않음

## 제작 절차
1. 힉스필드 create_website로 오손 랜딩 디자인 초안 생성
2. 초안 코드 확보 → `static/landing/`으로 이식 (외부 의존성 제거, 단일 HTML+CSS+JS)
3. landing 라우터 + leads 테이블 + 슬랙 알림 구현
4. 검증 후 배포 (main 푸시 → Railway 자동 배포)

## 검증
- 로컬 실행: 모바일(375px)·데스크톱 뷰 확인
- 폼 제출 → Supabase leads 저장 + 슬랙 알림 실측
- honeypot·rate limit 동작 확인
- 기존 라우트(/admin/, /kakao, /health) 회귀 없음 확인
